# 🐦 SuperGit-Jevi

**슈퍼하게 Git을 사용하는 제비처럼 빠른 Java TUI 도구**

## 왜 SuperGit-Jevi인가?

- **슈퍼깃**: Git을 슈퍼롭게 쓸 수 있어서 슈퍼깃
- **제비(Jevi)**: 제비처럼 빠르고 민첩하게! (원래는 자바(Java)였지만 제비가 더 좋아서 제비로 변신)
- **TUI**: 단순 CLI가 아닌 Terminal User Interface로 더 예쁘고 직관적인 경험!

## 특징

✨ **직관적인 TUI**: 컬러풀하고 예쁜 터미널 인터페이스 (Python의 Rich 같은 느낌)  
🎨 **박스형 UI**: 중요한 정보를 박스로 강조  
⚡ **제비처럼 빠른 실행**: JGit 기반의 빠른 Git 작업  
🛡️ **안전 장치**: **Push 전 자동 fetch+pull로 충돌 방지!**  
🚫 **인재 방지 시스템**: Pull 전에 Push를 막아주는 시스템 레벨 보호

## 핵심 안전 기능

### 🛡️ GitSafetyManager
Push를 시도할 때:
1. ✅ **자동 Fetch** - 원격 저장소 상태 확인
2. ✅ **자동 Pull** - 최신 변경사항 먼저 받기
3. ✅ **충돌 감지** - 병합 충돌 자동 감지
4. ✅ **안전 확인 후 Push** - 모든 검사 통과 시에만 Push

**절대 일어나지 않는 일**: Pull 없이 Push해서 팀원과 충돌하는 상황!

## 설치

```bash
# Maven으로 빌드
mvn clean package

# 실행 (별칭 설정 권장)
alias jevi='java -jar ~/개발프로젝트/Su\;pergit-jevi/target/supergit-jevi.jar'
```

## 사용법

### 기본 명령어

```bash
# 상태 확인 (아름다운 TUI로!)
jevi status

# 커밋 (슈퍼하게!)
jevi commit "메시지" -a

# 빠른 커밋 (타임스탬프 자동 추가)
jevi commit "작업 완료" -a -q

# 안전한 Push (자동 fetch+pull)
jevi push

# 강제 Push (안전 검사 무시, 권장하지 않음)
jevi push -f

# 브랜치 목록
jevi branch -l

# 새 브랜치 생성
jevi branch feature/awesome -c

# 브랜치 삭제 (확인 프롬프트 있음)
jevi branch old-feature -d

# 커밋 히스토리
jevi history

# 최근 5개 커밋만
jevi history -n 5

# 도움말
jevi --help
```

## 명령어 상세

### `jevi status`
- 저장소의 현재 상태를 TUI로 예쁘게 표시
- 파일 상태별 색상 구분 (수정/추가/삭제/충돌)
- 이모지로 직관적인 파일 상태 표시
- `-v`: 상세 정보 표시

### `jevi commit <message>`
- 스마트한 커밋 기능
- `-a, --all`: 모든 변경사항 자동 스테이징
- `-q, --quick`: 타임스탬프 자동 추가
- 커밋 전 변경사항 미리보기

### `jevi push`
**핵심 안전 기능!**
- 자동으로 fetch 수행
- 자동으로 pull 수행
- 충돌 감지 및 차단
- 모든 검사 통과 시에만 push 허용
- `-f, --force`: 안전 검사 무시 (경고 후 확인 필요)
- `-u, --set-upstream`: upstream 설정

### `jevi branch`
- `-l, --list`: 브랜치 목록 표시 (현재 브랜치 강조)
- `-c, --create`: 새 브랜치 생성
- `-d, --delete`: 브랜치 삭제 (확인 프롬프트)

### `jevi history`
- 커밋 히스토리를 테이블 형식으로 표시
- 해시, 메시지, 작성자, 날짜 정보
- `-n <number>`: 표시할 커밋 개수
- `-a, --all`: 모든 커밋 표시
